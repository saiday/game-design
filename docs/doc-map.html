Insignificant · Documentation Map

What each document is for, who reads it, and how they connect. Companion to the development dashboard.

Layout rule: one docs/ tree per level. Repo-root docs/ holds cross-project doctrine, process docs, and human dashboards. insignificant-game/docs/ holds the build contract and working docs. insignificant-game/design/ holds the game rules and is the single source of truth. Agent navigation lives in the two CLAUDE.md files (the root one carries the routing matrix); this page is the human-facing view of the same structure. Update both when a document moves or changes role.

Maintenance rules (the short version)

  • Current state only. Docs describe the present, written for fresh eyes. History lives in git commit messages. Exceptions that keep dates: PLAN.md, decisions.md, prompts.md, the cookbook §14 findings log, and license/verification records.
  • Single source of truth. Game rules live in design/ and nowhere else; other docs link instead of restating. The contract (architecture.md) fixes code shape, not rules.
  • Bidirectional metadata. Design frontmatter code: lists and module header citations move together, in the same change.
  • Skill-pinned paths. docs/adr/ and docs/agents/ are read by installed skills at those exact paths; never rename them.
  • Known absences are declared, not discovered. The routing matrix in the root CLAUDE.md distinguishes out-of-scope-by-design (multiplayer, save/load) from planned-but-pending (audio/music pipeline, localization workflow) so agents can tell intentional absence from missing docs.
  • When docs move or change role: update the root CLAUDE.md routing matrix and this page together, and bump both dashboards' footers.
